Solución al problema XOR¶

El problema XOR no puede ser solucionado por un perceptrn simple. Para esto, se introducen capas ocultas que ayudan a transformar los datos de entrada y que se puede determinar regiones de decisin adecuadas. Se ha probado que una red neuronal con una capa oculta es un aproximador universal, lo que quiere decir que, tericamente, es capaz de aproximar cualquier funcin de manera adecuada.

En este caso, damos algunas formas de solucionar el problema XOR utilizando capas ocultas, para tranformar los datos hacia un espacio en donde exista un separador lineal.

Funciones lógicas básicas¶

El problema XOR es un problema lgico complejo. Los problemas lgicos simples responden a las puertas NOT, AND y OR. Un problema lgico complejo implica que puede expresarse a partir de la combinacin de operadores lgicos simples. El XOR, por tanto, puede solucionarse con una composicin de los operadores simples. Por tanto, definimos primero estos operadores.

In [3]:
#Perceptrn AND
def AND(x):
    w = np.array([1,1])
    return 1*(np.dot(x,w)-1.5 > 0)

#Perceptrn OR
def OR(x):
    w = np.array([1,1])
    return 1*(np.dot(x,w)-0.5 > 0)

#Perceptrn NOT x_1
def NOT1(x):
    try:
        w = np.array([-1,0])
        return 1*(np.dot(x,w) + 0.5 > 0)
    except:
        return 1*(-x + 0.5 > 0)
    
#Perceptrn NOT x_2
def NOT2(x):
    try:
        w = np.array([0,-1])
        return 1*(np.dot(x,w) + 0.5 > 0)
    except:
        return 1*(-x + 0.5 > 0)

Estas funciones, entonces, clasifican los datos segn cada uno de los problemas.

Solución al problema XOR¶

Definimos una solucin al problema XOR de la siguiente forma:

$$XOR(x_1,x_2) = AND\Big( NOT\big(AND(x_1,x_2)\big), OR(x_1,x_2)\Big)$$

Se puede comprobar que esta composicin de funciones bsicas da como resultado el operador XOR.

Al aplicar esta solucin, podemos notar que se realiza una tranformacin a los datos; esta tranformacin est definida como:

$$h = \begin{pmatrix} NOT\big(AND(x_1,x_2)\big) \\ OR(x_1,x_2) \end{pmatrix}$$

El resultado de esta tranformacin son vectores dos dimensionales, pero en donde los puntos que pertenecen a la clase 1 colapsan en un solo punto $\begin{pmatrix} 1 & 1 \end{pmatrix}$ y los puntos de la clase 0 se mueven a las posiciones de los ejes del plano.

#transformacin
h = lambda x: np.array([NOT1(AND(x)), OR(x)]).T
#Funcin XOR
XOR = lambda x: AND(h(X))
print(XOR(X))

Podemos notar que la nueva configuracin de los datos se parece mucho a las posiciones que se dan en el problema AND. Por tanto, la capa de salida es un perceptrn para el problema AND.

Adems de esta solucin, podemos ver que el problema XOR tambin puede expresarse de la siguiente forma:

$$XOR(x_1,x_2) = OR\Big( AND\big( x_1, NOT(x_2) \big), AND\big( NOT(x_1), x_2 \big) \Big)$$

En este caso, la transformacin que se realiza a los datos es:

$$h = \begin{pmatrix} AND\big( x_1, NOT(x_2) \big) \\ AND\big( NOT(x_1), x_2 \big) \end{pmatrix}$$

Y la capa de salida es un perceptrn OR.

Mapeo a mayores dimensiones¶

Otra forma de solucionar el problema XOR es mapear los puntos a una mayor dimensin para encontrar all un hiperplano separador. Esto se puede hacer si asignamos una funcin $h: \mathbb{R}^2 \to \mathbb{R}^3$, que lleve los puntos a 3 dimensiones.

Asignamos los pesos adecuados, los bias, y las funciones de activacin. Los datos en 3 dimensiones podrn ser separados por una capa de salida que corresponde a un perceptrn simple.

In [9]:
#Pesos de capa oculta
w1 = np.array([[0.5, 1, 1],
              [0.5, -1, -1]])
b1 = np.array([0, 0, 0.5])
#Pesos de salida
w2 = np.array([0, 1.3, -1.6])
b2 = 0.6

#Capa oculta
h = lambda x: np.tanh(np.dot(x,w1) + b1)
#Capa de salida
f = lambda x: 1*(np.dot(h(x),w2) + b2 > 0)
print(f(X))

#Ploteo
fig = plt.figure(figsize=(7,7))
ax = plt.axes(projection='3d')
ax.scatter(h(X)[:,0], h(X)[:,1], h(X)[:,2], c=f(X), s=100)
ax.view_init(20,-50)
plt.title('Datos en el espacio 3d')
plt.show()
