Perceptrón¶

El algoritmo del perceptrn aprende los parmetros a partir de la iteracin a partir de una regla de actualizacin basada en una interpretacin del ngulo entre los vectores ejemplos y el vector de pesos. El perceptrn es una formalizacin de una neurona que procesa estmulos y emite una seal; se utiliza para realizar clasificacin (binaria generalmente).

El percpetrn es un clasificador lineal; es decir, se basa en encontrar un hiperplano que separe los datos en clases binarias (0 y 1) en base a la funcin:

$$f(x) = \begin{cases} 1 & \text{si } wx +b > 0 \\ 0 & \text{si } wx+b \leq 0 \end{cases}$$

#Perceptrn AND
def AND(x):
    w = np.array([1,1])
    return 1*(np.dot(x,w)-1.5 > 0)

#Perceptrn OR
def OR(x):
    w = np.array([1,1])
    return 1*(np.dot(x,w)-0.5 > 0)

#Perceptrn NOT x_1
def NOT(x):
    try:
        w = np.array([-1,0])
        return 1*(np.dot(x,w) + 0.5 > 0)
    except:
        return 1*(-x + 0.5 > 0)

Algoritmo del perceptrón¶

El perceptrn es un algoritmo de aprendizaje estadstico que busca estimar las clases de un conjunto de datos a partir de la funcin de decisin:

$$f(x; \theta) = \begin{cases} 1 & \text{ si } wx + b > 0 \\ 0 & \text{ si } wx + b \leq 0\end{cases}$$

Aqu $x$ es un vector de entrada. La funcin f depende de $\theta = {w,b}$ que representa los parmetros o los pesos de la red: $w$ contienen los pesos de las conexiones y $b$ es el bias. Definimos esta funcin a continuacin:

In [7]:
#Funcin de decisin
f = lambda x, theta: (np.dot(x,theta) >= 0).astype(np.int)

El objetivo del algoritmo del perceptrn es obtener los parmetros $\theta$ adecuados para que la red pueda clasificar adecuadamente los ejemplos. Este algoritmo es iterativo y busca minimizar el cunto se equivoca la red en cada paso.

Antes de entrar de lleno al este algoritmo, agregaremos una columna ms a la matriz de datos de entrenamiento; esto con el objetivo de que esta columna represente el bias, de tal forma que slo actualicemos sobre un vector de parmetros:

In [8]:
#Se agrega una columna para bias
x_train_bias = np.append(x_train, np.ones((x_train.shape[0],1)),axis=1)

Para inicializar el algoritmo del perceptron, tomaremos un vector $w \in \mathbb{R}^{n+1}$, donde $n$ es el nmero de variables (la dimensin de los vectores ejemplos) y el $+1$ responde al bias. Asimismo, estableceremos un nmero mximo de iteraciones (en caso de que el algoritmo no sea capaz de minimizar el error) y, finalmente, estableceremos un rango de aprendizaje.

In [9]:
#Parmetros
w = np.ones(x_train.shape[1] + 1)

#Nm. max de iteraciones
its = 100
#Learning rate
lr = 0.001

Para entrenar la red, utilizaremos el algoritmo del perceptrn. Este consta de dos partes:

  • Forward. Para cada ejemplo $x$, se computar la funcin $f(x; \theta)$
  • Backward. Se actualizarn los pesos de $\theta = {w,b}$ por medio de la regla: $$w_i \leftarrow w_i - \eta (f(x;\theta)-y_x)x_i$$ Donde $y_x$ es la clase a la que pertenence el ejemplo $x$.

El algoritmo iterar hasta que el error $f(x; \theta) -y$ sea 0 o cuando se cumpla el nmero mximo de iteraciones.

#Detenerse
stop = False
t = 0
#Evolucin del error
error = []
while  t < its:
    #Guradamos el error
    epoch_error = 0
    for x,y in zip(x_train_bias, y_train):
        #FORWARD
        #Calculamos la prediccin
        pred = f(x,w)
        
        #BACKWATD
        #Obtenemos eror f-y
        err = pred - y
        #Guardamos el error de la epoca
        epoch_error += err
        #Actualizamos los pesos
        w -= lr*err*x
    
    #Guarda los errores
    error.append((epoch_error**2).sum())
    #Aumenta la poca
    t += 1

Inferencia de probabilidades con perceptrón¶

El problema de de clasificacin puede verse como un problema de estimacin donde seleccionamos la probabilidad de clase ms alta. Esto se formaliza como:

$$\hat{y} = \arg\max_y p(Y=y|X_1,...,X_n)$$

Sin embargo, en el algoritmo del perceptrn no hemos estimado ninguna probabilidad, slo hemos decidido a partir de la fuerza del estmulo (si este es mayor a 0 se toma la clase 1, si es menor la clase 0). Con los parmetros aprendidos, podemos ver cmo se comportan los datos con respecto a esta decisin. Para esto mapearemos los escalares $wx+b$ en el eje $x$, mientras que en el eje $y$ se tienen las clases.

Los puntos de la clase 1 estn arriba, mientras que los de la clase 0 abajo. Los puntos ms cercanos a 0 son los ms dificiles de decidir. En este sentido, se tiene mayor certeza cuando el punto est ms alejado del 0, pues es claro que entre ms positivo sea un caso con ms certeza se puede asegurar que es de la clase 1. En sentido estricto, diremos que es ms probable que pertenezca a la clase 1. Por tanto, estipulamos que podemos definir una funcin de probabilidad que nos diga esto. Para esto, debemos buscar una funcin suave que aproxime la posicin de los datos. La funcin que buscamos es la funcin sigmoide, y esta dada por:

$$\sigma(x) = \frac{1}{1+e^{-x}}$$

Est funcin tiene las proiedades siguientes:

  • Es continua y derivable derivable
  • Converge a 1 cuando $x\to\infty$
  • Converge a 0 cuando $x\to -\infty$

En este sentido es una funcin con valores en $[0,1]$. Tomaremos al sigmoide como una funcin de probabilidad. En este sentido, la estimacin de probabilidad a aprtir del perceptrn puede verse como:

$$p(Y=1|x) = \sigma(wx+b)$$

Por su parte, la probabilidad de la clase negatica est dada por:

$$p(Y=0|x) = 1-\sigma(wx+b) = \sigma(-wx-b)$$

Esto euivale a proyectar los puntos obtenidos con la preactivacin del perceptrn $wx+b$ en la funcin sigmoide, como puede verse a continuacin: as, los puntos ms alejados de 0 positivamente quedarn en la seccin del sigmoide ms cercana a 1, mientras que los que se alejan negativamente quedarn en la seccin del sigmoide ms cercana a 0.

A continuacin mostramos esta proyeccin en los datos.

De esta forma, los puntos ms cercanos a 0 tendern a tener una prbabilidad cercana a $0.5$. De tal forma que podemos reformular la funcin de descin del perceptrn de la forma siguiente:

$$f_\sigma(x; \theta) = \begin{cases} 1 & \text{ si } \sigma(wx + b) > 0.5 \\ 0 & \text{ si } \sigma(wx) + b \leq 0.5\end{cases}$$

Est funcin tendr los mismos resultados que la funcin rgida de desicin. Pero esta funcin tiene la ventaja de ser derivable, por lo que se podrn utilizar mtodos de optimizacin basadas en gradiente. Por tanto, en adelante ocuparemos funciones de este tipo y no rgidas como las del perceptrn.

Como puede observarse, a partir de cambiar la funcin de activacin en la salida, se obtiene no valores discretos que representan una clase, si no valores probabilsticos. Se puede observar que si el valor es menor o igual a 0.5 es porque el ejemplo pertenece a la calse 0, de otro modo pertenece a la calse 1. A diferencia del perceptrn, este mtodo nos permite dar una probabilidad de que pertenezca a cada una de las clases. Para la clase 1, la probabilidad es $p(Y=1|X) = \sigma(wx+b)$. Para la clase 0, se tiene $p(Y=0|X) = 1-p(Y=1|X) = 1- \sigma(wx+b)$.

A este mtodo de estimacin de probabilidades se le conoce comnmente como regresin logstica.
