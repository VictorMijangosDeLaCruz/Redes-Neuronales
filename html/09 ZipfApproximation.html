En la teora de la informacin, un concepto central es la codificacin de un mensaje. Un mensaje es generado por una fuente, y este mensaje se codifica para ser enviado por el canal y que el receptor pueda decodificarlo y reconstruir el mensaje original. Generalmente la un mensaje se representa a partir de una variable aleatoria $X$. La informacin de esta variable tiene que ser codificada por un alfabeto. El alfabeto representa los smbolos que el canal puede transmitir. Por ejemplo, en teora de la computacin, se suele trabajar con un alfabeto binario, de 0s y 1s. Esta forma de codificar es comn. Pero tambin pueden existir otro tipo de codificaciones. Por ejemplo, el cdigo morse responde a un alfabeto formado por un punto (dot) $.$, una marca de mayor longitud $-$ (dash) y un espacio. Este alfabeto responde a las capacidades fsicas del canal de comunicacin.

Una codificacin $C$ para una variable aleatoria $X$ (mensaje) es un homomorfismo $C: X \to \Sigma^*$, tal que $\Sigma^*$ es el conjunto de cadenas (codewords) formadas a partir del alfabeto $\Sigma$.

La funcin $C$ de codificacin es un homomorfismo entre monoides; esto quiere decir que cumple que para cualesquiera $x_i, x_j \in X$ se tiene que $C(x_i x_j) = C(x_i) \cdot C(x_j)$. Como trabajaremos con alfabetos y cadenas, la operacin $\cdot$ representa la concatenacin. De esta forma, una codificacin de un conjunto de valores de una variable aleatoria es la concatenacin de cada uno de estos valores.

La codificacin ptima de un mensaje responde a diferentes factores: idealmente, buscamos que la codificacin permita una decodificacin eficiente. Pero tambin buscaremos que las longitudes de las cadenas no sean excesivas. Para esto, introduciremos el concepto de longitud esperada. Esta longitud esperada tiene una relacin particular con la entropa. Finalmente, veremos cmo estos conceptos pueden aplicarse al lenguaje, y a la estimacin de modelos del lenguaje (de orden 1).

Tipos de códigos¶

Supongamos que tenemos una variable aleatoria $X$ que puede tomar valores $x_1, x_2,..., x_N$. Por ejemplo, pueden representar edades, nombres o cualquier tipo de informacin que quiera transmitirse. Esta variable aleatoria representa un mensaje; este mensaje puede transmitirse a partir de un cdigo. Un cdigo estar determinado por un alfabeto $\Sigma$, a partir del cual buscaremos una forma eficiente de representar cada valor de la variable aleatoria. Un cdigo (o $codeword$) para un valor $x_i$ es una cadena $s \in \Sigma^*$. Aqu $\Sigma ^*$ representa el conjunto de todas las cadenas que se pueden formar a aprtir del alfabeto $\Sigma$. Podemos denotar el cdigo de $X=x_i$ como $C(x_i)$.

Para clarificar estos concepto, pongamos un ejemplo. Pensemos que tenemos una variable aleatoria $X$ representada por la pregunta ''De qu color es el gato?'', y supongamos que los posibles valores que puede tomar esta variable (las posibles respuestas) son: ''blanco'', ''amarillo'', ''gris'' y ''negro''. Ahora bien, si queremos transmitir esta informacin a aprtir de un cdigo binario, es decir, si contamos con un alfabeto $\Sigma =\{0,1\}$, entonces tenemos que elegir cadenas de 0s y 1s que puedan interpretarse como los colores que puede tener el gato.

Una opcin poco eficiente sera asignarle a todas las posibles respuestas a la variable $X$ el mismo smbolo; por ejemplo, todos los valores de la variable pueden estar representados por 0. As, parato todo valor $x_i$ de X, tendremos que $C(x_i) =0$. Sin embargo, esta codificacin es problemtica pues quien reciba el cdigo (el receptor) no podr saber a que color est refirindose, no podr decodificarlo. Este tipo de cdigo se conoce como cdigo singular, pues este tipo de codificacin asigna un mismo cdigo a ms de un slo valor del mensaje, lo que impide una decodificacin correcta.

Para que el cdigo pueda ser decodificado se requiere que sea no singular; es decir, que todo valor del mensaje tenga un cdigo diferente. Por ejemplo, una codificacin no singular para la variable $X$ que hemos propuesto podra ser:

  • $C(blanco) = 0$
  • $C(amarillo) = 010$
  • $C(gris) = 01$
  • $C(negro) = 10$

Aqu, cada valor de la variable tiene un cdigo distinto. Sin embargo, esta codificacin es problemtica cuando se enva ms de un valor. Por ejemplo, si queremos enviar un mensaje conjunto como $(gris, blanco)$, el cdigo que se enviara sera: $C(gris, blanco) = C(gris)C(blanco)=010$. Este cdigo es el mismo que se utiliza para codificar el valor $gris$. Por tanto, cuando se enve ms de un valor, el receptor tendr problemas con la decodificacin.

Para solucionar este problema, no basta con que cada valor del mensaje tenga un cdigo distinto; queremos, adems, que los cdigos de las cadenas de valores (que llamaremos $extensiones$) tengan tambin una codificacin distinta. Cuando esto pasa, se dir que la codificacin es unvocamente decodificable. Siguiendo con nuestro ejemplo, la siguiente codificacin es unvocamente decodificable:

  • $C(blanco) = 10$
  • $C(amarillo) = 00$
  • $C(gris) = 11$
  • $C(negro) = 110$

En este caso, podemos ver que $C(gris, blanco) = C(gris)C(blanco)= 1110$ que no coincide ya con ningn otro cdigo. En general se puede comprobar que, admes de diferenciar cada valor con un cdigo distinto, toda cadena de valores corresponder a una sola cadena de cdigos.

Existe una restriccin ms que podemos pedirle al cdigo. En el caso anterior $C(gris) =11$ es un prefijo de $C(negro) = 110$; es decir, el cdigo del valor $gris$ aparece como una subcadena (a la izquierda) del cdigo de $negro$. Si bien esto no impide que el receptor pueda decodificar adecuadamente el mensaje, genera una redundancia que rompe con la particin del cdigo (lo que puede ser problemtico en aplicaciones como los rboles de decisin). Idelamente, buscaremos que una codificacin donde ningn cdigo sea prefijo de otro. Una codificacin en que ningn cdigo es prefijo de otro se dice que es instantneo (o libre de prefijos). De nuevo, retomando nuestro ejemplo, la siguiente codificacin es instantnea:

  • $C(blanco) = 0$
  • $C(amarillo) = 10$
  • $C(gris) = 110$
  • $C(negro) = 111$

Esta codificacin permite particionar el espacio de los valores de manera eficiente. Por ejemplo, si el primer valor es 0, sabremos con certeza que el valor es $blanco$, dejando de lado todos los otros valores. Si el valor es $1$, por su parte, hemos descartado el valor $amarillo$ y tendremos que leer el siguiente bit: si es 0 sabremos que el valor es $amarillo$. Si es 1 tenemos que pasar al ltimo bit: ahora slo veremos si el ltimo bit es 0 o 1 para poder decidir el valor adecuado.

En resumen, tenemos los siguientes tipos de codificaciones:

1) singulares, donde ms un valor del mensaje tiene un mismo cdigo,

2) no singulares, donde cada valor del mensaje tiene un cdigo distinto,

3) unvocamente decodificables, donde cada valor y cada cadena de valores tiene una codificacin distinta,

4) instantneos, aquellos donde ningn cdigo es prefijo de otro.

Códigos óptimos¶

Una vez que hemos revisado los tipos de cdigos propuestos en la teora de la informacin, debemos determinar qu caractersticas debe tener una codificacin ptima.

Cuando se enva un mensaje, se busca que el cdigo enviado sea fcilmente decodificable; esto implica que el cdigo sea idealmente instantneo (o al menos unvocamente decodificable). Asimismo, es ideal que el cdigo sea lo ms corto posible. Es decir, buscamos la codificacin ms pequea que sea fcilmente decodificable.

La longitud del cdigo ser, entonces, un factor importante para la codificacin. Idealmente buscamos que la codificacin sea la ms corta posible. Idealmente, los elementos ms frecuentes podrn ser codificados a partir de un nmero menor de smbolos, mientras que elementos menos frecuentes requerirn de una longitud mayor. Es decir, la longitud de los cdigos dependern de las probabilides de los elementos que se codifican.

Sea entonces $p_i = p(X=x_i)$ la probabilidad de que el mensaje corresponda al elemento $x_i$, y sea $l_i = |C(x_i)|$ la longitud del cdigo para $x_i$ bajo la codificacin $C$. Por tanto, una probabilidad $p_i$ mayor implicar una longitud de cdigo $l_i$ menor y visceversa. Un ejemplo tpico de este uso de una codificacin ptima es el cdigo Morse. El cdigo Morse cuenta con un alfabeto que incluye los smbolos de punto '$\cdot$', lnea '$-$' y espacio ' '. El cdigo Morse busca codificar los caracteres utilizados en el idioma ingls, es decir, las letras 'a', 'b', 'c', hasta 'z'. En ingls la letra ms frecuente es la 'e', por lo que el cdigo para esta letra es el ms simple, el punto '$\cdot$'. Otra letra bastante frecuente es la 't', que se codifica con la lnea '$-$'. Las letras con mayor probabilidad tienen la menor longitud.

La forma en que podemos medir, en promedio, que tan adecuada es la codificacin en trminos de longitud es a partir de la longitud esperada. La longitud esperada, $L$, es la esperanza de las longitudes. sta se expresa como:

$$L = \sum_i p_i l_i$$

La longitud esperada es sumamente importante para la codificacin, pues buscaremos que, idealmente, esta sea lo ms pequea posible. De esta forma, los elementos con mayor probabilidad ($p_i$ alto) tendrn una longitud pequea ($l_i$ bajo) y los elementos con poca probabilidad, podrn codificarse con mayor longitudes. La longitud esperada es entonces un indicador de esto.

Como mencionamos, una forma ptima de codificacin es la de los cdigos instantneos. Este tipo de cdigos cumple una desigualdad, la desigualdad de Kraft, que los relaciona con la longitud de sus cdigos.

Desigualdad de Kraft: Para cualquier cdigo instantneo con alfabeto de tamao $N$ y longitudes $l_i, i=1,...,m$ se cumple la desigualdad:

$$\sum_{i} N^{-l_i} \leq 1$$

Demostrar esta desigualdad no es difcil y conlleva conceptualizar los cdigos instantneos de una forma bastante til. Considrese un rbol $N$-ario, esto es, un rbol donde cada nodo cuenta con $N$ hijos. Entonces los nodos representarn los cdigos $C(x_i)$ y las aristas representan los smbolos del alfabeto que bastan para componer el cdigo $C(x_i)$. El ejemplo ms sencillo sera tomar un alfabeto binario $\Sigma = \{0,1\}$; entonces determinaremos un rbol binario. Si quisieramos representar, por ejemplo, la codificacin binaria del ejemplo anterior para los colores del gato, tendramos una raz que se divide en dos hijos, el primero llegara a un nodo con el cdigo 0, correspondiente a $C(blanco)$, del otro lado se tendra una rama con un 1, que a su vez se dividira en dos ramas, una con un 0 que llevara al cdigo 10, y otra que debe dividirse en otras ramas para llegar a los cdigos faltantes 110 y 111.

Este ejemplo deja ver que, ya que estamos tratando con cdigos instantneos, no existe en el rbo $N$-ario ningn cdigo $C(x_i)$ que tenga hijos; es decir, todos los cdigos que representan a los valores de la variable aleatoria deben ser hojas del rbol. Ahora bien, si tomamos $l_{mx} = \max l(x)$ la longitud ms grande de los cdigos que tenemos. Si consideramos todos los cdigos al nivel $l_{mx}$ estaremos en la parte ms profunda del rbol; es decir, no abr cdigos que se encuentren ms abajo, aunque pueden haber cdigos al mismo nivel. En el ejemplo anterior $l_{ms} = 3$, pues los cdigos con mayor longitud (110 y 111) tienen longitud 3.

Si consideramos los cdigos a nivel $l_i$, veremos que una cadena a este nivel tendr $N^{l_{mx}-l_i}$ descendientes. Por ejemplo, con el ejemplo anterior, si consideramos $l_i=2$ (el nivel del cdigo 10), como tenemos un alfabeto binario tendremos $2^{3-2} = 2^1 = 2$ descendientes (que son precisamente los cdigos 110 y 111). Obsrvese que el nmero total de nodos en cualquier nivel es menor que $D^{l_{mx}}$, esto implica que si sumamos los descendientes en cada nivel, tendremos la desigualdad:

$$\sum_i D^{l_{mx}-l_i} \leq D^{l_{mx}}$$

Pero la parte de la izquierda puede reescribirse como $\sum_i D^{l_{mx}-l_i} = D^{l_{mx}} \sum_i D^{-l_i}$ y dividiendo entre $D^{l_{mx}}$ de ambos lados de la desigualdad obtenemos la desigualdad de Kraft.

Esta desigualdad es importante pues tambin su inverso es verdadero; es decir, si tenemos un conjunto de longitudes de cdigos $l_1,...,l_m$ que cumple la desigualdad de Kraft, podemos construir un cdigo con esas longitudes. La estrategia para construir este cdigo se basa en la construccin del rbol que hemos utilizado para la demostracin del resultado.

Ahora podemos combinar las dos formulaciones que tenemos (la longitud esperada y la desigualdad de Kraft) para buscar una codificacin ptima. Queremos que la longitud esperada $L$ sea lo ms pequeo posible, sujeto a que se cumpla la desigualdad de Kraft (para garantizar que podremos encontrar una codificacin instantnea). Utilizando multiplicadores de Lagrange, podemos definir una funcin objetivo como:

$$J = \sum_i p_i l_i + \lambda \sum_i N^{-l_i}$$

El objetivo, entonces, ser encontrar las longitudes $l_1, l_2,..., l_m$ que minimicen esta funcin objetivo. tendremos que diferenciar con respecto a $l_i$ y as obtendremos las derivadas parciales:

$$\frac{\partial J}{\partial l_i} = p_i - \lambda N^{-l_i} \log N$$

Si igualamos estas derivadas a 0 y despejamos $N^{-l_i}$, tendremos que:

$$N^{-l_i} = \frac{p_i}{\lambda \log N}$$

Podemos tomar $\lambda = \frac{1}{\log N}$ de tal forma que obtenemos que $N^{-l_i} = p_i$. Si despejamos ahora $l_i$ de esta ltima ecuacin, obtendremos que la longitud ptima est dada por:

$$l_i^* = -\log_N p_i$$

Esta frmula refiere a la informacin. Por ejemplo, si el alfabeto es binario ($N=2$) la longitud ptima es $l_i^* = -\log_2 p_i$ bits. Ms an, si tenemos una codificacin ptima, donde todos las longitudes son $-\log_N p_i$, entonces tenemos que la longitud esperada es:

$$L^* = \sum_i p_i l_i^* = -\sum_i p_i \log_N p_i = H_N(X)$$

Es decir, la longitud esperada de un cdigo ptimo es igual a la entropa (en base $N$) de la variable aleatoria que se esta codificando.

Si bien muchas veces no se podr alcanzar un cdigo ptimo, pues para esto se requiere que $-\log_N p_i$ sea un entero. En su defecto, podramos tomar el entero menor que sea ms grande que $-\log_N p_i$. Pero de este resultado podemos obtener un corolario interesante. Ya que la longitud esperada es igual a la entropa si y slo si las longitudes de la codificacion minimizan esta longitud, entonces tenemos que la entropa es una cota inferior para la longitud esperada. Esto es:

$$H_N(X) \leq L$$

Slo si el cdigo es instantneo y las longitudes son iguales a $-\log_N p_i$ se tendr la igualdad.

Distribución de Zipf¶

Se conoce como Ley de Zipf a una ley emprica del lenguaje que nos dice que la frecuencia $f$ de una palabra $w$ es inversamente proporcional a su rango estadstico. El rango estadstico es la posicin que la palabra ocupa en una lista de acuerdo con su frecuencia. As, la palabra ms frecuente tendr el rango 1, la segunda ms frecuente el rango 2, etc. Por ejemplo, en el CREA, la palabra 'de' tiene rango 1, mientras que la palabra 'que' tiene rango 3. Entonces la Ley de Zipf nos dice que:

$$f(w_r) \propto r^{-\alpha}$$

donde $f(w_r)$ es la frecuencia de la palabra con el rango $r$. Adems $\alpha$ es un parmetro de ajuste, que tendr gran importancia y revisaremos ms adelante.

Antes de pasar a otra cosa, debemos mencionar es que una de las propiedades interesantes de la Ley de Zipf es que la escala logartmica 'lineariza' los datos. Esto es claro, pues si obtenemos el logaritmo de ambos lados de la ecuacin, tendremos que:

$$\log f(w_r) = -\alpha \log r + \beta$$

Recordemos que la proporcionalidad $\propto$ implica que existe una constante que al multiplicar el lado derecho da la igualdad. Es decir, $f(w_r) \propto r^{-\alpha}$ implica que existe una constante $c$ tal que $f(w_r) = c r^{-\alpha}$. En la ecuacin logartmoca, entonces, $\beta = \log c$ es una traslacin. El menos indica que la pendiente de la recta es negativa, es decir, que la funcin es descendiente.

En los datos del CREA podemos ver que la Ley de Zipf empricamente. La siguiente grfica muestra que, en efecto, hay la relacin del logartmo de las frecuencias y el logartmo del rango es descendiente.

Si bien, no se describe una recta perfecta, se puede ver que los datos empricos pueden aproximarse a aprtir de una recta. De hecho, la frmila que hemos determinado con los logaritmos $\log f(w_r) = -\alpha \log r + \beta$, se parece mucho al problema de la regresin lineal si tomamos $y=\log f(w_r)$ y $x=\log r$ tenemos, en efecto, un problema de regresin lineal. Resolver este problema de regresin lineal, de hecho, nos dar el valor de $\alpha$.

De esta forma, podemos interpretar a $\alpha$ como la pendiente de la recta descrita por la Ley de Zipf, o bien como una parmetro de ajuste para que a partir de los rangos podamos encontrar la frecuencia. Resolvamos entonces el probelma de regresin lineal a partir de los mnimos cuadrados. Tendremos entonces una funcin objetivo dada por:

$$J_{MSE} = \sum_r (\log f(w_r) - (-\alpha \log r + \beta))^2$$

En este caso, $\beta$ es conocido y corresponde al logaritmo de la frecuencia ms alta; es decir, $\beta = \log fr(w_1)$, este valor permite que la recta determine de manera precisa la frecuencia de la palabra con el rango 1, pues cuando $r=1$ entonces $\log r = 0$, por lo que $-\alpha \log 1 + \log f(w_1) = -\alpha 0 + \log f(w_1) = \log f(w_1)$. De esta forma, al menos podemos estimar la frecuencia de la palabra de rango 1 con error 0.

Cuando hablamos de la distribucin de Zipf, precisamente hacemos referencia a una distribucin estadstica que parte de la Ley de Zipf. En general, la probabilidad de una palabra con rango $r$, que expresamos como $p_r = p(X=r)$ est dada por:

$$p_r = \frac{r^{-\alpha}}{\sum_{k=1}^m k^{-\alpha}}$$

Donde $m$ es el nmero de palabras diferentes. Generalmente a $\sum_{k=1}^m k^{-\alpha}$ se le llama funcin de particin y puede verse como una versin finita de la funcin zeta de Riemann.

Otras distribuciones¶

Esta no es la nica forma en que podemos estimar las probabilidades de las palabras, una forma muy comn es a partir de una funcin de probabilidad frecuentista (la frecuencia relativa), donde la probabilidad est dada como:

$$p_r = \frac{f(w_r)}{\sum_k f(w_k}$$

Esta es una forma muy conocida de calcular probabilidades. Sin embargo, existe tambin otras propuestas para estimar la probabilidad de palabras.

Benoit Mandelbrot, por ejemplo, extiende la Ley de Zipf a la que se conoce como Ley de Mandelbrot-Zipf o simplemente Ley de Mandelbro. Esta es una generalizacin de la Ley de Zipf, que nos dice que:

$$f(w_r) \propto \frac{B}{(r+\rho)^\alpha}$$

Esta ecuacin ajusta mejor a los datos empricos pues el termino $(r+\rho)$ permite cierta curvatura. Asimismo, si $B=1$ y $\rho=0$ obtendremos la Ley de Zipf como la hemos enunciado anteriormente.

De nuevo, a partir de los datos del CREA podemos ver como se comporta la aproximacin de Mandelbrot con respecto a los datos empricos. Elijamos entonces $B=3$ y $\rho=2$. Tomamos $\alpha = 1.181227$, como en el caso de Zipf. La siguiente grfica muestra tanto la aproximacin de Mandelbrot como la de Zipf.

De igual forma que con Zipf, podemos obtener una funcin d eprobabilidad a partir de la funcin de Mandelbrot. Esta estar dada por:

$$p_r = \frac{B/(r+\rho)^\alpha}{Z}$$

Donde $Z = \sum_k \frac{B}{(k+\rho)^\alpha}$ es la funcin de particin.

Generalmente a la estimacin de estas probabilidades se les conoce como modelos del lenguaje. Un modelo del lenguaje consiste en la descripcin de un lenguaje natural a partir de probabilidades. La idea de los modelos del lenguaje es aproximar probabilidades de cadenas a partir de la factorizacin de sus elementos. En este caso, diremos que nuestra aproximacin es de orden 1. Es decir, estimamos probabilidades de palabras asumiendo independencia. Para Shannon (1948) existen aproximaciones de mayor orden: una aproximacin de orden 2 aproxima las probabilidades a partir de secuencias de dos palabras, y se conoce comunmente como modelo de bigramas.

En nuestro caso, estimar la probabilidad de una sola palabra ya parece complicado, especialmente cuando tenemos diferentes formas de estimar la probabilidad de la palabra. Cul de estas funciones de probabilidad ser mejor? Para responder esto podemos hacer uso de los conceptos de la teora de la informacin.

Estimación de una distribución que minimice la longitud esperada¶

Como hemos mencionado, para la teora de informacin un cdigo ptimo es aquel que minimiza la longitud esperada. Tambin hemos mencionado que esperamos que se cumpla la desigualdad de Kraft para asegurar que tenemos una codificacin instantnea. Sin embargo, ya hemos visto que el lenguaje natural no es un cdigo instantneo. De hecho, para los datos del CREA tenemos que:

$$\sum_r N^{-l_r} = 1.330995$$

En el caso de las estimacin de probabilidades de palabras la longitud esperada puede ser un buen indicador, y la funcin de Zipf una buena aproximacin, que adems resulta fcil de estimar.
